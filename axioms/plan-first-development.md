# Plan-First Development

**Statement**: No coding without an approved plan.

## Derivation

Coding without a plan leads to rework and scope creep. Plans ensure alignment with user intent before investment.

## Evidence

Evidence of violations and corrections tracked in bd issues (label: `learning`).

## Enforcement

Hard gate via EnterPlanMode tool before coding. See [[RULES.md]].
