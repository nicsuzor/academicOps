# Planning Workflow Index

Two workflows for the [[planning]] skill's three operating modes.

| Workflow             | Mode   | When to Use                                             |
| -------------------- | ------ | ------------------------------------------------------- |
| [[strategic-intake]] | Mode 1 | New idea, constraint, or fragment to place in the graph |
| [[decompose]]        | Mode 2 | Validated epic/goal needs breaking into concrete tasks  |

## Routing

- **Mode 1 (Strategic Intake)**: User brings a fragment. Use `strategic-intake.md`.
- **Mode 2 (Epic Decomposition)**: Epic is ready for concrete work. Use `decompose.md`.
- **Mode 3 (Prioritisation)**: No specific workflow — use graph tools directly (`get_network_metrics`, `get_dependency_tree`, `pkb_orphans`).
