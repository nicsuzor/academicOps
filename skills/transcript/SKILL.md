# Transcript Skill

Convert Claude Code JSONL session files to readable markdown transcripts.

## Quick Start

```bash
PYTHONPATH=$AOPS uv run python $AOPS/scripts/claude_transcript.py <session.jsonl> -o <output-base>
```

## Output

Two versions generated by default:
- `<output>-full.md` - Complete transcript with tool results in collapsible sections
- `<output>-abridged.md` - Compact transcript showing tool calls without results

Use `--full-only` or `--abridged-only` for single version.

## Standard Naming

Output files should follow: `YYYYMMDD-<project>-<slug>-{full,abridged}.md`

Output location: `$ACA_DATA/sessions/claude/`

## Finding Session Files

```bash
# Current session (most recent in current project)
ls -lt ~/.claude/projects/-$(basename $(pwd))/*.jsonl | grep -v agent | head -1

# List all recent sessions
ls -lt ~/.claude/projects/*/[0-9a-f]*.jsonl | grep -v agent | head -10
```

## Examples

```bash
# Generate both versions with standard naming
PYTHONPATH=$AOPS uv run python $AOPS/scripts/claude_transcript.py ~/.claude/projects/-home-nic-src-aOps/abc123.jsonl \
  -o $ACA_DATA/sessions/claude/20251203-aops-restructure

# Full version only
PYTHONPATH=$AOPS uv run python $AOPS/scripts/claude_transcript.py session.jsonl --full-only

# Abridged version only
PYTHONPATH=$AOPS uv run python $AOPS/scripts/claude_transcript.py session.jsonl --abridged-only
```

## Workflow

1. Find the session JSONL file (use commands above)
2. Determine project name and descriptive slug
3. Run script with standardized output path
4. Verify output in `$ACA_DATA/sessions/claude/`

## Architecture

Uses `lib/session_reader.py` which parses and combines:
- Main session JSONL (`*.jsonl`)
- Agent transcripts (`agent-*.jsonl`)
- Hook logs (`*-hooks.jsonl`)

Same module used by the dashboard skill for live activity display.
