# Transcript Skill

Convert Claude Code JSONL session files to readable markdown transcripts.

## Quick Start

```bash
cd $AOPS && uv run python scripts/claude_transcript.py <session.jsonl> -o <output-base>
```

## Output Versions

Two versions generated by default:

| Version | Content | Use Case |
|---------|---------|----------|
| `*-full.md` | Full injected content (fenced), tool results | Deep analysis, debugging |
| `*-abridged.md` | File references only, no tool results | Quick review, daily summary |

Use `--full-only` or `--abridged-only` for single version.

## Standard Naming

Output files: `YYYYMMDD-<project>-<slug>-{full,abridged}.md`

Output location: `$ACA_DATA/sessions/claude/`

## Finding Session Files

```bash
# Current session (most recent in current project)
ls -lt ~/.claude/projects/-$(basename $(pwd))/*.jsonl | grep -v agent | head -1

# List all recent sessions
ls -lt ~/.claude/projects/*/[0-9a-f]*.jsonl | grep -v agent | head -10
```

## Examples

```bash
# Generate both versions with standard naming
cd $AOPS && uv run python scripts/claude_transcript.py \
  ~/.claude/projects/-home-nic-src-aOps/abc123.jsonl \
  -o $ACA_DATA/sessions/claude/20251203-aops-restructure

# Full version only
cd $AOPS && uv run python scripts/claude_transcript.py session.jsonl --full-only

# Abridged version only
cd $AOPS && uv run python scripts/claude_transcript.py session.jsonl --abridged-only
```

## Architecture

| Component | Role |
|-----------|------|
| `scripts/claude_transcript.py` | CLI entry point |
| `lib/session_reader.py` | Parses JSONL, formats markdown |

Session reader combines:
- Main session JSONL (`*.jsonl`)
- Agent transcripts (`agent-*.jsonl`)
- Hook logs (`*-hooks.jsonl`)

Same module used by dashboard skill for live activity display.
