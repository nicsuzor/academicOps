# Design-First, Not Constraint-First

**Statement**: Start from "what do we need?" not "what do we have?" Current state is not a constraint.

## Derivation

Constraint-first thinking limits innovation. Design-first enables optimal solutions.

Evidence of violations and corrections tracked in bd issues (label: `learning`).
